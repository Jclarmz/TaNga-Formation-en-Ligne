PHP & MySQL

👨 Je vais ici vous apprendre ces diverses notions : Créer une base de données, Lier PHP à MySQL, Les Requêtes, qui constituent tout le chapitre sur PHP et MySQL.

Créer une base de données
😕 Avant de créer une base de données, Monsieur Declau, pouvez-vous nous dire ce que c'est s'il vous plaît ?

Ce que nous avons avant fait jusqu'à présent, c'est choisir d'utiliser les sessions pour stocker nos messages. Le problème est qu'une session n'est accessible que par l'initiateur de ladite session. Elle ne permet donc pas à notre interlocuteur d'accéder aux messages que nous lui avons envoyés et vice versa. Nous optons donc pour les bases de données, car elles offrent un système central auquel chaque correspondant a accès en lecture et en écriture, c'est-à-dire que chaque correspondant va y sauvegarder l'ensemble de ses messages avec la possibilité de lire tous les messages qui se trouvent dans la base de données.

😕 Comment lier PHP à une base de données, Monsieur Declau ?

Il existe plusieurs possibilités permettant de lier PHP à une base de données, dont la fameuse extension PDO, disponible depuis PHP5. Malheureusement, nous n'allons pas en parler ici !

Pour lier notre chat à une base de données, nous utiliserons l'extension mysqli petite sœur de mysql qui a été largement répandue par PHP4 et est aujourd'hui obsolète ; ce qui n'est pas le cas de mysqli.
Commençons par créer une base de données que nous allons appeler db_chat (voir le cours sur phpMyAdmin).
À la base de données db_chat, créons une table nommée message ayant deux champs : idMessage et valeurMessage.

Vidéo sur la création de la base de données db_chat et de la table message
Lier PHP à MySQL

Pour accéder à une base de données, il faut connaître :
✓ le nom et mot de passe d'un utilisateur ayant les droits d'acces à la base de données. Comme nous somme en local, pour faire simple, nous allons utiliser l'utilisateur tout puissant root qui n'a pas de mot de passe `` ;
✓ le nom de l'hôte où se trouve la base de données (pour notre cas, notre hôte est notre ordinateur donc localhost) ;
✓ le nom de la base de données (notre base de données s'appelle db_chat).

La connexion se fait en deux étapes :
✓ ouvrir la connexion vers MySQL : l'ouverture se fait grâce à la fonction mysqli_connect. Elle prend en paramètre l'hôte, le nom d'utilisateur et le mot de passe.
On écrit : mysqli_connect(nom_hôte, nom_utilisateur, mot_de_passe);

nom_hôte, nom_utilisateur, mot_de_passe peuvent être soit des variables, soit des constantes mysqli_connect retourne true si la connection s'est bien passée et false sinon.

✓ sélectionner la base de données : elle se fait grâce à la fonction mysqli_select_db*. Elle prend deux paramètres la connexion retourner par *mysqli_connect et le nom de la base de données.
On écrit : mysqli_select_db(connexion, nom_de_la_base_de_données);

nom_de_la_base_de_données peut être soit une variable, soit une constante

Il ne nous reste plus qu'à ajouter ces deux instructions dans index.html, juste après l'instruction session_start(); Ce qui donne : 

                
                    <?php
                      session_start();
                      /* Etablissons la connexion vers MySQL */
                      $cnx = mysqli_connect('localhost', 'root', '');
                      /* vérifions que la connexion 
                      C’est bien passée avant de continuer */
                      if(!$cnx) die('Problème de connexion au serveur');
                      /* sélectionnons notre base de données */
                      mysqli_select_db($cnx, 'db_chat');

                      /* ... */
                    ?>
Les requêtes

Une fois la connexion créée, il ne nous reste plus qu'à émettre des requêtes vers la base de données pour y enregistrer ou lire des messages. La communication avec une base de données se fait dans un langage spécifique le SQL (Structured Query Langage). Sans trop nous étendre sur le sujet, nous n'allons étudier que deux types de requêtes SQL :

✓ Requête d'insertion des données : elle permet d’insérer un nouvel enregistrement dans l'une des tables de la base données à laquelle nous somme connectés avec un compte utilisateur ayant les droits d’accès en écriture. Ce qui est le cas de l'utilisateur root.
On écrit : INSERT INTO nom_de_la_table (champ_1, champ_2, ..., champ_n) VALUES (valeur_champ_1, ..., valeur_champ_n);

NB :
Les champs auto_incrément (case A_I du champ coché) doivent prendre la valeur NULL, car c'est le système qui détermine leur valeur. Le champ champ_1, champ_2, ..., champ_n doivent tous être des champs de la table nom_de_la_table et les valeurs valeur_champ_1, ...,valeur_champ_n doivent correspondre au type du champ auquel il est associé.

✓ Requêtes de sélection : elles permettent de sélectionner et de récupérer (lire) des enregistrements venant d’une ou plusieurs tables de la base données à laquelle nous somme connectés avec un compte utilisateur ayant au moins les droits d’accès en lecture.
On écrit : SELECT champ_1, champ_2,..., champ_n FROM nom_de_la_table WHERE condition

Les champs champ_1, champ_2, ..., champ_n doivent tous être des champs de la table nom_de_la_table. Le WHERE condition est facultatif. On peut ordonner l'ensemble des valeurs sélectionnée en ajoutant ORDER BY nom_champ ASC ou ORDER BY nom_champ DESC pour respectivement ordonner nos enregistrements dans l'ordre croissant et décroissant du champ nom_champ. Order BY se place après WHERE condition

Pour exécuter une requête, on utilise la fonction mysqli_query, qui prend pour paramètre la requête à envoyer à la base de données.
On écrit : mysqli_query(connexion, ma_requete);

connexion est la valeur renvoyée par mysqli_connect.
ma_requete peut être une requête de lecture (sélection) ou une requête d'écriture (insertion, update ou autre).
ma_requete est une variable ou une constante de type chaine de caractère (texte entre guillemet " " ou entre côte ' ').
Lorsque ma_requete est une requête de lecture, mysqli_query renvoie un identifiant de résultat en cas de succès, et en cas d’échec, quelque soit le type de requête, mysqli_query renvoie false.

Exemple :
Code : 

                
                      <?php
                          /* ... */

                          /* Cette requête sélectionne l'identifiant et 
                          la valeur de tous les messages
                          de la table message sans condition */
                          mysqli_query($connexion, 'SELECT idMessage, valeurMessage FROM message');
                          /* Cette requête sélectionne l'identifiant et 
                          la valeur selement des messages dont le champ idMessage>5 */
                          mysqli_query($connexion,'SELECT idMessage, valeurMessage FROM message WHERE idMessage>5');
                          /* Cette requête sélectionne enregistre le message
                          Bonjour à tous dans la table message */
                          mysqli_query($connexion,'INSERT INTO message (idMessage, valeurMessage) VALUES (NULL, 'Bonjour à tous!');');

                          /* ... */
                      ?>

Afin d'obtenir un tableau PHP, l'identifiant de résultat retourné par mysqli_query doit être traité par la fonction mysqli_fetch_assoc. Cette dernière prend l'identifiant de résultat comme paramètre et retourne un tableau associatif, tel que la valeur de chaque attribut est accessible dans le tableau par le nom de cet attribut.

Nous allons donc modifier notre code de sorte que les sauvegardes ne se fassent plus dans les sessions, mais dans la base de donné db_chat, de même que pour les lectures. Nous allons donc retirer toutes références aux sessions tout en gardant la mention session_start(); pour permettre au système de nous identifier de façons unique sur le serveur.
Le code devient : 

                
                    <?php
                      session_start();
                      /* Etablissons la connexion vers MySQL */
                      $cnx = mysqli_connect('localhost', 'root', '');
                      /*vérifions que la connexion 
                      C’est bien passée avant de continuer*/
                      if(!$cnx) die('Problème de connexion au serveur');
                      /* sélectionnons notre base de données */
                      mysqli_select_db($cnx, 'db_chat');
                      /* je vérifie kil y a aucun 
                      new message envoyé */
                      if(isset($_POST['msg'])){
                      /* Si il y a un message envoyé
                      je le traite pour éviter l'injection HTML 
                      et je le garde dans la base de donnée */ 
                      $nouveau_msg = htmlentities($_POST['msg']);
                      mysqli_query($cnx, "INSERT INTO message (idMessage, valeurMessage) VALUES (NULL, '$nouveau_msg');");
                      }
                      //initialiser la liste des messages
                      $liste_messages = array();
                      /*récupérer l'ensemble des messages 
                      enregistrés dans la base de données*/
                      $resultat=mysqli_query($cnx, "SELECT idMessage, valeurMessage FROM message;");
                      /* Vérifier que la requete c'est bien passé si c'est le cas
                      Transformer l'identifiant de résultat en tableau PHP*/
                      if($resultat)
                      for($i=0; $ligne=mysqli_fetch_assoc($resultat); $i++) $liste_messages[] = $ligne;
                      else echo mysqli_error($cnx);
                    ?>

Et mettre toujours dans index.html le code ci-dessous entre les balises div (dans la zone d'affichage).
Code : 

                
                    <?php
                      /*enfin j'affiche l'ensemble des messages 
                      extrait de ma base de données*/
                      for($i = 0; $i < count($liste_messages); $i = $i+1){
                          $liste_messages[$i];
                      }
                    ?>

À ce niveau, il est déjà possible d'échanger des messages entre deux correspondants, car l'intermédiaire entre eux à savoir la base de données est bien en place et fonctionnelle. Pourtant, ce code présente de nombreux problèmes entre autre ceux liés à la sécurité, mais aussi au design. Donc, en particulier pour des raisons de sécurité, le code ci-dessus ne doit jamais être utilisé dans l'état.
